1.4 快捷键

  • ctrl + a //光标移到开头
  • ctrl + e //光标移到结尾
  • ctrl + c //强制终止当前命令
  • ctrl + l //清屏
  • ctrl + u //删除或剪切光标之前的命令
  • ctrl + k //删除或剪切光标之后的命令
  • ctrl + y //粘贴剪切的内容
  • ctrl + r //在历史命令中搜索
  • ctrl + d //退出当前终端
  • ctrl + z //暂停并放入后台
  • ctrl + s //暂停屏幕输出
  • ctrl + q //恢复屏幕输出

1.5 前后台作业控制

  • 命令 & //直接让作业在后台运行
  • ctrl + z //将当前作业切换到后台
  • jobs //查看后台作业状态
  • fg %n //让后台运行的作业切换到前台
  • bg %n //让指定的作业在后台运行
  • kill %n //移除指定的作业n

1.6 输出重定向

  • /dev/null ： 指垃圾箱
类型 符号 作用
标准输出重定向 command > file 以覆盖的方式把命令的正确结果输出到文件中
… command >> file 以追加的方式把命令的正确结果输出到文件中
标准错误输出重定向 command 2 > file 以覆盖的方式把命令的错误结果输出到文件中
… command 2 >> file 以追加的方式把命令的错误结果输出到文件中
正确错误同时保存 command > file 2 > &1 以覆盖的方式把命令的正确和错误结果同时输出到文件中
… command > file1 2 > file2 以覆盖的方式把正确结果输出到file1，把错误结果输出到file2

1.7 输入重定向

符号 说明
command < file 将file文件的内容作为命令输出
command << END 从标准输入（键盘）中读取数据，直到遇到分界符END才停止（分界符可自定义）

1.8 管道| 以及| tee

| tee 指输出到屏幕的同时还保存到文件中,如果tee指定的文件已存在，那么会覆盖文件的内容
例如：cat A.txt | tee A.txt

1.9 通配符

字符 含义 实例
* 匹配0个或者多个任意字符 a*b,a与b之间可以有任意长度的字符，也可以没有。例如:aabcb,ab,azxcb…
? 匹配一个任意字符 a?b,a与b之间必须但也只能存在一个字符，该字符可以是任意字符。例如: aab, abb,acb…
[list] 匹配list中的任意单个字符 a[xyz]b，a与b之间必须但也只能存在一个字符，该字符只能是x或y或z。例如: axb,ayb,azb
[!list] 匹配除list中的任意单个字符 a[!a-z]b，a与b之间必须但也只能存在一个字符，该字符不能是小写字母。例如: aAb，aOb…
[c1-c2] 匹配c1-c2间的任意单个字符 a[0-1]b,a与b之间必须但也只能存在一个字符，该字符只能是数字。例如: a0b,a1b…
{string1,string2,…} 匹配string1、string2中的一个字符串 a{abc,xyz,opq}b,a与b之间必须但也只能存在一个字符串,字符串只能是abc或xyz或opq。例如: aabcb，axyzb,aopqb…

2.2 作业脱机管理

1.将作业(进程)切换到后台可以避免由于误操作如[ctrl]+c等导致的job被异常中断的情形，而脱机管理主要是针对终端异常断开的情形。
⒉通常使用nohup命令来使得脱机或注销之后，Job依旧可以继续运行。也就是说nohup忽略所有挂断(SIGHUP)信号。
3.如果该方式命令之后未指定&符号，则job位于前台，指定&符号，则job位于后台。

2.3 screen

  • -A 将所有的视窗都调整为目前终端机的大小。
  • -d <作业名称> 将指定的screen作业离线。
  • -h <行数> 指定视窗的缓冲区行数。
  • -m 即使目前已在作业中的screen作业，仍强制建立新的screen作业。
  • -r <作业名称> 恢复离线的screen作业。
  • -R 先试图恢复离线的作业。若找不到离线的作业，即建立新的screen作业。
  • -s 指定建立新视窗时，所要执行的Shell.
  • -S <作业名称> 指定screen作业的名称。
  • -v 显示版本信息。
  • -x 恢复之前离线的screen作业。
  • -ls或–list 显示目前所有的screen作业。
  • -wipe 检查目前所有的screen作业，并删除已经无法使用的screen作业。

3.变量

3.1 自定义变量

  • 定义变量： 变量名=变量值 其中变量名必须以字母或者下划线开头，区分大小写，例如：name=limerence dir=pwd
  • 引用变量：$变量名
  • 查看变量：echo $变量名 set
  • 取消变量：unset 变量名
  • 作用范围：仅在当前shell中有效

3.2 环境变量

  • /etc/profile
  • ~/.bash*
  • 定义变量：方法一 export name=limerence 方法二 export 变量名 将自定义变量转换为环境变量
  • 查看变量：echo $变量名 env

3.3 位置变量

假如在weizhi.sh中写入echo "$1"和echo “$2” ， 执行bash weizhi.sh 1 2 ,则两个都会输出，但是如果执行bash weizhi.sh 2 ， 结果是第一个输出为2，第二个输出为空。

3.4 预定义变量

  • $$ : 指当前进程的PID

  • $? : 指上一个命令的返回值 0表示成功

5.函数

5.1 定义函数

  • function 是 shell 中的关键字，专门用来定义函数；
  • name 是函数名
  • commands是函数要执行的代码，也就是一组语句;
  • return value表示函数的返回值，其中return是Shell关键字，专门用在函数中返回一个值;这一部分可以写也可以不写。
  • 由{}包围的部分称为函数体，调用一个函数，实际上就是执行函数体中的代码。

5.2 调用函数

5.3 for循环

  • for 每次从值集合中取一个值赋值给变量。
  • do - done将赋值后的变量带入执行的命令得到执行结果。
  • 重复以上两个步骤，直到值集合中的值被—一获取赋值给变量的到所有结果，循环结束。

5.4 while循环

  • while 首先进行条件测试，如果传回值为O(条件测试为真)，则进入循环，执行命令区域，否则不进入循环
  • 满足while测试条件，执行命令区域，直到while的测试条件不满足结束执行while循环(如果条件一直满足执行无穷循环)。

5.5 until循环

  • until 条件测试结果为假（传回值不为0)，就进入循环。
  • 条件测试不满足，执行命令区域。直到until 条件满足，结束执行until循环(如果条件一直不满足则执行无穷循环)。

5.6 循环控制

命令 说明
break n n表示跳出循环的次数，如果省略n表示跳出整个循环
continue n n表示退到低n层继续循环，如果省略n表示跳过本次循环进入下一次循环
exit n 退出当前的shell程序，返回n，n也可以省略
return 用于返回一个退出值给调用的函数
shift 用于将参数列表list做义务指定次数，最左端的那个参数就从列表中删除，后面的参数继续进入循环

6.符号说明

6.1 函数常用符号

  • (()) 数值比较，运算。适用C语言
  • [[]] 条件促使，支持正则
  • $(()) 整数运算
  • $[] 整数运算
  • $() 命令替换，shell会先执行括号的cmd，然后将结果作为变量进行替换，替换只能替换标准输出，错误输出不能替换
  • ${} shell中变量的原型，用于限定变量名称的范围，并且支持通配符
  • [] 条件测试
  • () 子shell中执行
  • {} 在当前shell执行
  • 注意：
    • ()最后一个命令可以不用分号，{}最后一个命令要用分号
    • ()第一个命令和左边括号不必有空格，而{}第一个命令和左括号必须有一个空格
    • ()和{}里的某个命令重定向只影响改命令，而括号外的重定向则影响到括号里的所有命令

6.2 文件测试表达式

符号 作用
-d文件 文件存在且为目录则为真
-f文件 文件存在且为普通文件则为真
-e文件 文件存在则为真，不辩别是目录还是文件
-s文件 文件存在且文件大小不为0则为真
-r文件 文件存在且可读则为真，与执行脚本的用户权限也有关
-w文件 文件存在且可写则为真，与执行脚本的用户权限也有关
-x文件 文件存在且可执行则为真，与执行脚本的用户权限也有关
-L文件 文件存在且为链接文件则为真
f1 -nt f2 文件f1比文件f2新则为真，根据文件的修改时间计算
f1 -ot f2 文件f1比文件f2旧则为真，根据文件的修改时间计算

6.3 字符串测试表达式

符号 参数 功能
-z s1 如果字符串s1的长度为0，则测试条件为真
-n s1 如果字符串s1的长度大于0，则测试条件为真
sl 如果字符串s1不是空字符串，则测试条件为真
=或== s1=s2 如果s1等于s2，则测试条件为真,=前后应有空格
!= s1!=s2 如果s1不等于s2，则测试条件为真
< s1 如果按字典顺序s1在s2之前，则测试条件为真
> s1>s2 如果按自定顺序s1在s2之后，则测试条件为真

6.4 整数测试表达式

在[]和test中使用 在[[ ]]和(())中使用 说明

8.数组

8.1 数组操作

8.1.1 数组定义

  • 方法一：用小括号将变量值括起来赋值给数组变量，每个变量值之间要用空格分隔，例如 S=（1 2 3）
  • 方法二：用小括号将变量值括起来，同时采用键值对的形式赋值，例如 S=（[0]=one [1]=two [3]=three）
  • 方法三：通过分别定义数组变量的方法定义，例如 S[0]=one
  • 方法四：动态的定义数组变量，并使用命令的输出结果作为数组的内容，例如 S=($(命令)) S=(命令)

8.1.2 数组使用

  • ${ARRAY_NAME[INDEX]} 引用数组中的元素

  • ${ #ARRAY_NAME[*]} 数组中元素的个数

  • ${ #ARRAY_NAME[@]} 数组中元素的个数

  • ${ARRAY_NAME[@]} 引用数组中的所有元素

  • ${ARRAY_NAME[*]} 引用数组中的所有元素

  • ${ #ARRAY_NAME} 数组中下标为0的字符个数

  • echo ${array[1]} 数组打印

  • unset array[] 删除数组

  • echo ${array[@]:1:3} 从下标为1开始截取到3结束

  • echo ${array[@]:2:A} 把数组中的2替换为A，注意原数组未修改

  • echo ${array[@]#p*} 从左边开始最短匹配p，并删除。%是右边

  • echo ${array[@]##p*} 从左边开始最长匹配p，并删除。%%是右边

10. 正则表达式

字符 BRE/ERE 含义
. BRE&ERE 匹配任意单个字符（除字符串结束符NUL）
^ BRE&ERE 匹配行首，如^abc，匹配以abc开头的字符串
$ BRE&ERE 匹配行尾
* BRE&ERE 匹配0个或任意多的单个字符，前置字符可以是正则表达式
+ ERE 匹配前面正则表达式的1个或多个实例
? ERE 匹配前面正则表达式的0个或1个实例
[…] BRE/ERE 匹配括号内任一字符，常配合-字符使用，表示匹配一个连续的范围。^字符作为方括号内的第一个字符表示匹配不在方括号内的任意字符
- BRE/ERE 连字符
{n,m} BRE 区间表达式，表示匹配在它前面的字符n到m次。其中，n与m的值必须介于0-RE_DUM_MAX（含）之间，后者最小值为255
{n} BRE 表示匹配在之前的字符n次
{n,m} BRE 功能同上
{n} BRE 功能同上
\ BRE&ERE 转义符
() ERE 匹配位于方括号内的正则表达式
() BRE 将\之间的模式保存在特殊的“保留空间”中，最多可以存储9个，可以通过后续的转义蓄力\n来匹配”保留空间“中的模式
\n BRE 与（）结合起来使用，\1匹配第一个子模式、\2匹配第二个，，最多到\9
竖杠 ERE 匹配位于竖杠符号前或后的正则表达式
